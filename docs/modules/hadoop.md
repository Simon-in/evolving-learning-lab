# Hadoop 组件学习文档

## 1. 组件定位

Hadoop 是离线大数据体系的基础底座，核心解决两类问题：

- 存储：用 HDFS 存放大规模数据，支持分布式、副本容错。
- 资源调度：用 YARN 管理集群资源，承载 Hive/Spark/Flink 等计算任务。

在你的学习路线中，Hadoop 的价值是：先建立“存储 + 调度”基础认知，后续学 Hive/Spark/Flink 才有统一坐标系。

## 2. 核心概念

### 2.1 HDFS（存储层）

- NameNode：管理元数据（目录树、文件到 block 的映射）。
- DataNode：真正存放数据块（block）的节点。
- Block：大文件会拆成多个块分布到不同节点。
- Replication：每个 block 多副本，提升容错能力。

你至少要能回答：

1. 文件上传到 HDFS 后，元数据和数据分别存在哪？
2. 为什么 HDFS 默认适合大文件、顺序读写？
3. 副本机制如何影响可靠性和成本？

### 2.2 YARN（资源层）

- ResourceManager：全局资源管理与调度中心。
- NodeManager：单机资源管理，负责容器生命周期。
- Application：一次作业提交（例如 Spark/Hive 任务）。
- Container：任务运行资源单元（CPU/内存）。

你至少要能回答：

1. 一次作业从提交到运行，YARN 做了什么？
2. 为什么同一套 YARN 能承载多种计算引擎？
3. 队列和资源隔离在生产环境中的意义是什么？

## 3. 本阶段实操任务（最小闭环）

建议用最小闭环来验证学习效果，而不是只看概念。

### 任务 A：HDFS 基础操作

- 创建目录、上传文件、下载文件、查看文件内容。
- 目标：熟悉 `put/get/ls/cat` 等基础命令行为。

### 任务 B：验证数据分布与副本

- 观察文件 block 分布和副本信息。
- 目标：把“分布式存储”从抽象概念变成可观察事实。

### 任务 C：YARN 作业观测

- 提交一个最小任务并在 YARN UI 观察生命周期。
- 目标：理解 Application、Container、日志与失败排查入口。

## 4. 常见问题与排查

### 4.1 HDFS 写入失败

常见原因：
- NameNode 未就绪
- DataNode 不可用或磁盘不足
- 副本数要求高于可用节点数

排查思路：
- 先确认服务状态，再看 NameNode/DataNode 日志，最后检查副本策略配置。

### 4.2 YARN 任务 Pending 或失败

常见原因：
- 资源不足（内存/CPU）
- 队列限制
- 任务参数不合理

排查思路：
- 先看 YARN UI 的 Application 状态，再看容器日志定位具体报错。

- HDFS 思维 -> 对象存储思维（S3 / OSS / COS / OBS）

重点不是记产品名，而是理解“存储抽象 + 资源抽象”。
