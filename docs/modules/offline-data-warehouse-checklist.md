---

## 一、必做任务（按优先级）

- [ ] 任务 1：确定业务主题与指标口径（5-10 个核心指标）
- [ ] 任务 2：完成分层模型设计（ODS-DWD-DWS-ADS + 主键/分区/更新策略）
- [ ] 任务 3：准备可复现样例数据（含空值、重复、迟到等脏数据场景）
- [ ] 任务 4：实现 Hive DDL 与分区装载脚本
- [ ] 任务 5：实现 Spark 离线加工链路（至少 1 条）
- [ ] 任务 6：接入调度（Airflow 或 DolphinScheduler 最小 DAG）
- [ ] 任务 7：补数据质量校验（空值率、主键重复、分区完整性、指标波动）
- [ ] 任务 8：做一次性能优化并记录前后对比
- [ ] 任务 9：沉淀“面试可讲”项目文档（背景-方案-难点-优化-结果）

---

## 二、阶段验收标准（DoD）

完成以下 5 条后，离线数仓阶段可视为达标：

1. 能跑通 `调度 -> Hive/Spark -> ADS 宽表` 全链路。
2. 能清晰讲解分层建模与指标口径设计。
3. 有性能优化证据（耗时/扫描量/参数前后对比）。
4. 有基础数据质量校验与失败处理说明。
5. 有一份可用于面试讲解的完整离线项目复盘。

---

## 三、面试表达检查点

- [ ] 能解释为何先做离线闭环，再做实时与湖仓。
- [ ] 能说明 Hive 与 Spark 在链路中的职责分工。
- [ ] 能展示一次真实问题排查与优化过程。
- [ ] 能用“通用能力 + 云上映射”表述项目经验（避免 AWS 术语绑定）。
